Detailed Description

Strategy transaction state.

When strategy starts an operation it is said to be in transaction; for instance, to pause a strategy could mean to delete a child strategy, which is a (potentially) remote operation and operation result will not be available immediately – in this case, strategy should set its transaction state to "Pending", and when operation is finished, state should be set to "Ok" or "Failed", depending on operation result.

It is important to understand that it is fully user responsibility to update transaction state accordingly depending on operation progress.
