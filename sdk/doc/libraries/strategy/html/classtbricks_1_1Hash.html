Detailed Description

template<typename KeyT, typename ValueT>
class tbricks::Hash< KeyT, ValueT >

Hash template.

Current implementation is based on dense hash table, which is very efficient in time, but consumes in average up to 4x times more memory than required to store inserted elements. In general prefer SparseHash, use this Hash only if you need the table on a super critical path.
